# 웹 성능 최적화 점검 보고서

요청하신 체크리스트를 기반으로 현재 시스템(`WooriDo`)의 성능 최적화 상태를 점검했습니다.

## 📊 종합 결과: 최적화 단계 도입 시급 (Early Stage)
대부분의 최적화 설정이 아직 적용되지 않은 초기 개발 단계입니다. 아래 항목들을 다음 스프린트에 반영하시길 권장합니다.

---

## 1. 전송 효율화 (Transfer Efficiency)

| 항목 | 상태 | 현재 현황 | 개선 제안 |
| :--- | :---: | :--- | :--- |
| **파일 압축** | ❌ 누락 | Frontend/Backend 모두 Gzip/Brotli 설정 없음 | **FE**: `vite-plugin-compression` 추가<br>**BE**: `server.compression.enabled=true` 설정 |
| **이미지 최적화** | ❌ 누락 | 원본 이미지 사용 중 (빌드 시 변환 없음) | **FE**: `vite-plugin-imagemin` 도입 또는 CDN 이미지 리사이징 사용 |
| **HTTP/2, 3** | ❌ 누락 | 기본 HTTP/1.1 사용 (SSL 인증서 필요) | 운영 배포 단계에서 Nginx/AWS ALB 레벨에서 HTTP/2 적용 필요 |
| **CDN** | ❌ 누락 | 로컬 환경 위주 | AWS CloudFront 등 CDN 도입 검토 |

## 2. 부하 분산 & DB

| 항목 | 상태 | 현재 현황 | 개선 제안 |
| :--- | :---: | :--- | :--- |
| **Caching (Redis)** | ❌ 누락 | `build.gradle`에 Redis 의존성 없음 | **Redis 도입 시급**. Refresh Token 저장소 및 자주 조회되는 챌린지 데이터 캐싱용 |
| **Batching** | ❓ 확인 필요 | 명시적인 배치 처리 로직 부재 | 로그 기록, 조회수 증가 등은 `@Async` 또는 배치로 분리 |
| **304 응답** | ⚠️ 기본 | Spring Boot 기본 설정에 의존 | ETag 필터 설정 확인 및 API 응답에 대한 명시적 캐싱 적용 |

## 3. 클라이언트 최적화

| 항목 | 상태 | 현재 현황 | 개선 제안 |
| :--- | :---: | :--- | :--- |
| **Bundling** | ⚠️ 기본 | Vite 기본 설정 사용 | `manualChunks` 설정으로 벤더(React 등) 파일 분리 (Chunk Splitting) |
| **Cache Policy** | ❌ 누락 | 정적 파일 캐싱 헤더 설정 없음 | 배포 시 Nginx 또는 S3 메타데이터에 `Cache-Control` 설정 |
| **Initial Load** | ⚠️ 주의 | 최적화 부재로 JS 번들 사이즈 큼 | 코드 스플리팅(`React.lazy`) 적용 및 불필요한 라이브러리 제거 |

## 🚀 우선순위 추천 (Next Action)

1.  **[Backend] Redis 도입**: JWT 보안(로그아웃 구현)과 성능(데이터 캐싱) 두 마리 토끼를 잡기 위해 가장 시급합니다.
2.  **[Both] Gzip 압축 적용**: 설정만으로 전송 데이터 크기를 70% 이상 줄일 수 있는 가성비 좋은 작업입니다.
3.  **[Frontend] 빌드 최적화**: Vite 플러그인을 추가하여 JS/CSS 압축 및 이미지 변환을 자동화하세요.
